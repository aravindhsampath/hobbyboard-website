Documentation

Hobbyboard is a self-hosted visual archive designed for makers, designers, and digital hoarders. It combines the power of AI-assisted organization with the privacy of local storage.

Installation

We offer three ways to install Hobbyboard: Docker (recommended), Pre-built Binaries, or building from Source.

Method 1: Docker Compose

The easiest way to get started. Requires Docker Desktop or Docker Engine.

Terminal
curl -O https://raw.githubusercontent.com/aravindhsampath/hobbyboard/main/docker-compose.yml
docker compose up -d

Method 2: Binary

Download the latest release for your platform from the GitHub Releases page. You will also need to install ffmpeg and libheif dependencies manually.

Terminal
# macOS
brew install ffmpeg libheif
./hobbyboard serve

Configuration

Hobbyboard uses a hobbyboard.toml file for settings. When you first run the app, a default file is generated.

Key Settings

  • raw_media_dir

    Path to your source images. Read-only access is sufficient.

  • dist_dir

    Where thumbnails and database files are stored.

  • ai_provider

    Select between ollama (local), openai, or gemini.

Usage Workflow

Hobbyboard works in a cycle of Scan -> Process -> Explore.

  1. Scan: Point the app to your folder. It identifies new files.
  2. Build: The AI vision model captions your images, while local models generate embeddings for search.
  3. Serve: The web interface allows you to browse, search, and curate.

AI Models

You can choose between local privacy and cloud accuracy.

Local (Ollama)

Completely offline. Requires decent GPU/RAM. Slower but private.

Cloud (OpenAI/Gemini)

Fast, high accuracy. Requires API key. Data is sent to provider for analysis only.

Troubleshooting

If something goes wrong, check the logs in the web interface or your container logs.

Terminal
docker compose logs -f app
